1. Prepare Files

RE-FOLDR requires two files. Both must be loaded before any action (Classify, Export XML, Export Unassigned) is available.

FileWhere to get it
Plugin Report.txtCubase → Studio → VST Plug-in Manager → Plug-in Report button at the bottom of the window.
PluginManager.xmlCubase's plugin collection file. Must be created with Add All Plug-ins for full plugin ID coverage — see below.
C:\Users\[사용자]\AppData\Roaming\Steinberg\Cubase [버전]\PluginManager.xml

Why Add All Plug-ins matters

Each plugin needs a unique ID (cid) in the exported XML for Cubase to recognize it. The Plugin Report lists all installed plugins but contains no IDs. The only reliable source of IDs for all plugins — including shared-container plugins like Waves — is a PluginManager.xml created via New Collection → Add All Plug-ins in Cubase's VST Plug-in Manager.

Steinberg bundled plugins (EQ, Compressor, Reverb, HALion Sonic SE, Groove Agent SE, Padshop, Retrologue, and others included with Cubase) are recognized automatically — their IDs are built into RE-FOLDR and do not require any file. Paid full editions (e.g. HALion 7, Groove Agent 5) are separate plugins with their own IDs and are read from PluginManager.xml.
Back up your original PluginManager.xml before replacing it with RE-FOLDR's export. RE-FOLDR does not modify the original file, but keep a copy in case you need to revert.

2. Load Files

Drag and drop files onto the RE-FOLDR window — one at a time or both at once, in any order. The app detects each file type automatically.

Until both files are loaded, the collections and plugin panels are hidden and replaced by a checklist showing which files are still missing.

Once both files are loaded, the checklist disappears and the full interface appears. Any existing collections from PluginManager.xml are imported automatically.

You can reload or replace either file at any time by dropping a new version onto the window. RE-FOLDR re-merges everything each time.

3. FX and Instruments

RE-FOLDR separates plugins into two tabs at the top of the right panel.

TabPlugin type
FXEffects — EQ, Dynamics, Reverb, Delay, Modulation, and more.
InstrumentsVSTi — Synth, Keys, Drum, Bass, Player, Orchestra, Band, Vocal, Utility.

Each tab has its own collections, folders, and classification engine. The left panel always shows collections for the current tab.

4. Workflow

The typical RE-FOLDR workflow is:

  1. Load both files. Existing collections from Cubase appear in the left panel.
  2. Click + New Collection to create a new collection. For FX, choose a folder order (Category sorts by signal processing purpose; Name A–Z sorts alphabetically; Count sorts by plugin count after classification). For Instruments, folders are always created in A–Z order.
  3. Click Create & Classify — RE-FOLDR creates the default folders and runs automatic classification immediately.
  4. Review results in the right panel. Reassign plugins using drag-and-drop or the folder dropdown on each row.
  5. Click Export XML when done. Replace PluginManager.xml in Cubase's data folder with the downloaded file.
The Classify button in the toolbar re-runs classification on the active collection at any time — useful after updating RE-FOLDR to a version with an improved dictionary. Manual assignments are always preserved.

5. Collections

A collection is a named set of folders that appears as a tab in Cubase's Plug-in Manager.

Activate

Click a collection name in the left panel. The active collection is highlighted. All Classify and folder operations apply only to the active collection.

Rename

Double-click a collection name to edit it inline. Press Enter or click elsewhere to confirm, Esc to cancel.

Create

Click + in the Collections header. Enter a name and (for FX) choose folder order. Click Create & Classify to build folders and run classification immediately, or Skip to create an empty collection.

Delete

Hover a collection row and click the trash icon. Plugins in that collection become Unassigned.

6. Folders

Filter by folder

Click a folder in the left panel to filter the plugin list to that folder. Click again to deselect.

Reorder folders

Drag any folder row to reorder within the same collection. A line indicator shows the insertion point. Reordering a folder automatically switches the sort mode to Manual, and the export preserves this order.

Add a folder

Click Add folder at the bottom of a collection. Choose Auto (participates in Classify, supports keywords) or Manual (drag-and-drop only, never touched by Classify).

Rename a folder

Double-click a folder row to edit the name inline. Press Enter or click elsewhere to confirm, Esc to cancel.

Subfolders

Click the + icon on the right of a folder row to add a subfolder. If subfolders exist, the icon toggles collapse/expand instead.

Delete a folder

Hover a folder row and click ×. Plugins in that folder become Unassigned.

Dragging plugins onto Add folder creates a new folder and assigns the selected plugins in one step.

7. Classify

The Classify button assigns plugins to folders automatically using a three-pass engine. Runs on the active collection only.

PassMethodDescription
1DictionaryExact name lookup in a built-in table (12,000+ FX entries, 1,700+ instrument entries).
2Name ruleKeyword matching against built-in rules and any custom keywords set on Auto folders.
3Cubase typeUses the plugin's declared type from the Plugin Report (e.g. Fx|EQ, Instrument|Synth).

Each pass only processes plugins left unassigned by the previous one. Plugins without a matching rule remain Unassigned.

Re-running Classify resets all auto-assigned plugins in the active collection and re-classifies from scratch. Manually assigned plugins (via drag or dropdown) are preserved. A confirmation dialog appears if any plugins are already assigned.

When to use Classify again

The typical flow runs Classify once via Create & Classify. Use the Classify button again when:

  • You've updated to a newer version of RE-FOLDR with an improved dictionary
  • You've added or modified folder keywords and want to re-apply them
  • You want to reset auto-assigned plugins and start over

Color legend

Each plugin row shows a colored dot indicating how it was assigned.

Amber — Dictionary (Pass 1)
Orange — Name rule (Pass 2)
Green — Cubase type (Pass 3)
Blue — Brand-prefix guess (lowest confidence)
Red — Unassigned

8. Assign Plugins

Single plugin — dropdown

Click the folder button on the right of any plugin row to open a dropdown of available folders. Select a folder to assign, or type a new folder name to create one.

Single plugin — drag

Drag a plugin row from the right panel and drop it onto a folder in the left panel.

Multiple plugins

  1. Click to select a plugin. Shift+click selects a range. Ctrl+click (or Cmd+click on Mac) toggles individual items.
  2. Drag any selected row to a folder, or use the Move to dropdown in the action bar.

Copy to folder (Favorites)

Hold Ctrl (or Cmd) while dropping onto a folder to copy the selected plugins there without removing them from their current folder. Use this to maintain a Favorites folder alongside your main classification.

Cubase's behavior with duplicate plugin entries in the same collection is unverified. Test the Favorites copy feature with one plugin before applying widely.

Unassign

Select plugins and click Unassign in the action bar.

Export Unassigned

Click Unassigned in the toolbar to export unclassified plugins as JSON (name, vendor, subcategory only — no file paths). Useful for reporting missing plugins to help expand the dictionary.

Manual assignments are preserved when Classify is re-run.

9. Sort & Reorder

Plugin sort

Use the sort dropdown in the toolbar to change how plugins are listed in the right panel.

ModeBehavior
Name A–ZAlphabetical by plugin name.
VendorGrouped by vendor, then by name within each vendor.
ManualPreserves the order set by drag-and-drop. Export uses this order.

Reorder plugins within a folder

With a folder selected in the left panel, drag plugin rows in the right panel to reorder them. The sort mode switches to Manual automatically. Export preserves this order.

Reorder folders

Drag any folder row in the left panel to reorder it within its collection. The sort mode switches to Manual automatically.

The exported XML always reflects the current sort mode — Manual order is written as-is; A–Z and Vendor orders are applied at export time.

10. Display Name

Display Name sets a short label written to the exported XML — useful for keeping Cubase mixer insert slot labels readable.

Example: FabFilter Pro-Q 3 → FF Pro-Q3

Edit individually

Double-click the plugin name in the right panel to enter inline edit mode. Press Enter or click elsewhere to confirm, Esc to cancel. Clear the field to remove the display name.

Auto DN

Select one or more plugins and click Auto DN in the action bar. RE-FOLDR generates abbreviated names based on vendor initials and product tokens.

Cubase matches plugins by ID (cid), not by name. Display Name only affects how the plugin appears in the Plug-in Manager list. Test with one plugin before applying to your full library.

11. Export XML

Click Export XML in the toolbar. RE-FOLDR downloads PluginManager.xml containing all collections and folder assignments from both FX and Instruments tabs.

Before downloading, RE-FOLDR checks for two issues: plugins not assigned to any folder (won't appear in any exported collection), and plugins assigned to a folder but missing a cid (excluded from export). If either is found, a warning shows the counts and offers Export anyway. If neither exists, Export runs immediately.

Load into Cubase

  1. Make sure Cubase is fully closed before replacing the file.
  2. Locate the original PluginManager.xml and replace it with the exported file.
  3. Reopen Cubase → Studio → VST Plug-in Manager. Your collections should appear.
Always replace with Cubase fully closed. If Cubase is open when the file is replaced, it will overwrite the file with its own version on exit — losing your changes.
